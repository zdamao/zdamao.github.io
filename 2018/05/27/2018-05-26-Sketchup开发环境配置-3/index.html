调试设置

完成本步骤前，需要先完成Sketchup开发环境配置-Rubymine项目配置 (2)

准备Sketchup

配置Rubymine调用Sketchup。

从github下载sketchup-ruby-debugger文件，安装步骤请阅读下载包内的Readme。

准备IDE

接下来配置Rubymine，借助一点点启动脚本，我们可以从Rubymine启动Sketchup的调试模式。

启动脚本

在根目录下创建一个tools文件夹，下载文件debug-sketchup.rb，或者手动新建并拷贝以下代码。

目前项目结构截图如下：

调试配置

访问菜单：Run > Edit Configurations…

调试Sketchup中的Ruby代码，需要配置remote debugging：

  1. 可以同时配置多个版本的Sketchup调试；
  2. Rubymine不知道Sketchup是开启还是关闭的，所以我们勾选”Single instance”, 这样当我们关闭或重启Sketchup时，Rubymine停止监听上一次开启的Sketchup；
  3. 确保端口和debug-sketchup.rb里的配置一致；
  4. Remote root folder路径设置为Sketchup的插件路径。
  5. Local root folder路径同样设置为Sketchup的插件路径。

配置Rubymine启动Sketchup

将Sketchup配置为外部工具（External Tool)。


通过给调用的debug-sketchup.rb增加参数，可以控制使用哪个版本的Sketchup。

注意我们将debug-sketchup.rb配置为相对路径，也可以设置为绝对路径，但不够灵活。

调试

开始调试，从调试工具栏选择配置项。

点击右侧的调试按钮，也可以从菜单项选择Run > Run

Rubymine开始监听Sketchup，当Sketchup以调试模式开始后，Sketchup会显示为冻结状态，直到Rubymine连接到Sketchup。

Rubymine连接到Sketchup后可以开始调试，尝试设置断点。

切换到Sketchup开始测试，设置了断点的Sketchup会停止，在Rubymine中对断点进行操作。

通过Rubymine的调试窗口，尝试对断点进行操作。

Rubymine会在代码窗口显示变量值。

Rubymine有一个窗口列出了所有当前项目中的变量。

如果你关闭了Sketchup，当你再次通过Rubymine重新调用Sketchup时，窗口会提示你已经有一个进程在开启状态，这是因为Rubymine不知道你已经关闭了Sketchup。因为之前我们勾选了”Single instance”，它会在每次调用Sketchup时都提示我们关闭之前的进程，并新建一个进程。如果你没有勾选”Single instance”，Rubymine可能不能成功调用Sketchup，因为端口仍然被上一个进程占用。
